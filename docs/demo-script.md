# 演示脚本（上传 / 下载 / 简化模式）

本文用于现场演示与手工验收，目标是让观众在 10-15 分钟内看清以下能力：

## 文档职责

- 负责：演示步骤、预期结果、异常排查与验收记录模板。
- 不负责：组件 API 详解与架构原理展开。

- 自动上传、自动清理
- 自动网络策略（弱网/断网恢复）
- IndexedDB 持久化与刷新续传
- 简化模式的“清单 + 弹窗上传 + 实时刷新”
- 下载分片、暂停恢复、刷新后继续

## 1. 演示前准备

1. 安装依赖

```bash
npm install
npm --prefix server install
```

2. 启动后端

```bash
npm run server:dev
```

3. 启动前端（新终端）

```bash
npm run dev
```

4. 打开页面

- 前端：`http://127.0.0.1:5173`
- 后端：`http://127.0.0.1:3000`

5. 准备演示文件（建议）

- `small-demo.txt`（几十 KB）
- `medium-demo.mp4`（20-100 MB，便于观察分片和进度）

## 2. 场景 A：完整上传模式（自动能力主演示）

### A-1 基础自动流程

1. 切到“上传”Tab。
2. 选择 `small-demo.txt` 与 `medium-demo.mp4`。
3. 观察无需手动点击“上传”即可开始（`autoUpload=true`）。

预期结果：

- 文件进入队列并自动上传。
- 进度条持续更新，最终状态为“已完成/秒传成功”。
- 批次结束后出现自动清理倒计时（若开启 `autoCleanup`）。

### A-2 断网恢复与自动续传

1. 上传中途在浏览器 DevTools 将网络切到 `Offline`（或断网）。
2. 观察上传停止/等待。
3. 恢复网络。

预期结果：

- 上传任务继续执行，不需要重新选文件。
- 仅补传缺失分片，不从 0% 全量重传。

### A-3 刷新后续传（IndexedDB）

1. 在大文件上传未完成时刷新页面。
2. 回到“上传”Tab。

预期结果：

- 组件检测并恢复中断任务（基于 IndexedDB）。
- 在 `autoUpload=true` 且网络正常时自动继续上传。

## 3. 场景 B：简化模式（集成友好演示）

### B-1 清单 + 弹窗上传

1. 切到“简化模式”Tab。
2. 确认页面主体是“服务器文件清单”。
3. 点击“上传文件”按钮，弹出上传对话框。
4. 在弹窗内选择文件，观察自动上传。

预期结果：

- 弹窗内仅保留“选择文件 + 进度列表”，无复杂高级操作。
- 单文件上传成功后，主界面清单立即刷新。
- 批次结束后弹窗自动关闭。

### B-2 文件名语义校验

1. 上传一个原始文件名明显的文件（如 `user-visible-name.png`）。
2. 观察简化模式清单中的文件名。

预期结果：

- 清单展示原始文件名，不展示物理存储 hash 名。

## 4. 场景 C：下载模式（断点下载演示）

1. 切到“下载”Tab，确认列表可见。
2. 选择一个较大文件点击下载。
3. 下载过程中点击暂停，再点击继续。
4. 下载未完成时刷新页面后继续下载。

预期结果：

- 下载分片可暂停/恢复。
- 刷新后能从断点继续（IndexedDB 分片缓存仍在）。
- 下载完成后可导出文件。

## 5. 可选场景 D：SQLite 可视化核验

如需展示后端落盘，可使用 `DB Browser for SQLite`：

- 下载地址：https://sqlitebrowser.org/dl/
- 默认数据库：`server/data/indexeddb-upload.sqlite`

建议在停止后端后再打开数据库，查看：

- `files`
- `file_chunks`

## 6. 快速异常排查

1. 列表未刷新：先点“刷新”，再检查后端是否正常、`/api/file/list` 是否返回 `code=200`。
2. 上传卡住：检查网络状态与浏览器控制台；确认后端 `upload/merge` 接口可达。
3. 文件名显示 hash：重启后端并确认已使用最新代码，重新拉取列表。
4. 下载无法续传：检查浏览器是否清理了站点存储（IndexedDB）。
